# Plan 02-02: OpenSky Flight Tracking with OAuth2

## Goal
Add live commercial flight tracking to the globe with ~6,700 aircraft updating every 15 seconds, rendered as rotated airplane icons with BillboardCollection.

## Tasks

### Task 1: OpenSky OAuth2 Token Manager
**Files:** `backend/src/services/opensky-auth.ts` (new)

Implement OAuth2 client credentials flow:
1. Token endpoint: `https://auth.opensky-network.org/auth/realms/opensky-network/protocol/openid-connect/token`
2. `grant_type: client_credentials`, `client_id` + `client_secret` from env vars
3. Cache token in memory with expiry tracking (tokens last 30 min)
4. Auto-refresh when token is within 2 minutes of expiry
5. Env vars: `OPENSKY_CLIENT_ID`, `OPENSKY_CLIENT_SECRET`
6. Graceful fallback: if no credentials, use unauthenticated access (limited to 400 credits/day)

### Task 2: OpenSky Fetcher
**Files:** `backend/src/fetchers/opensky.ts` (new)

Extend BaseFetcher:
- `sourceId: 'flights'`
- `defaultInterval: '*/15 * * * * *'` (every 15 seconds)
- `cacheTTL: 12` (slightly less than interval)
- `fetchRaw()`: GET `https://opensky-network.org/api/states/all` with OAuth2 bearer token
- `normalize()`: Convert state vector arrays to GeoJSON FeatureCollection
  - Filter out entries with null lat/lon
  - Map array indices to named fields: icao24, callsign, origin_country, lat, lon, baro_altitude, velocity, true_track, vertical_rate, on_ground, squawk
  - Convert velocity from m/s to knots, altitude from meters to feet
- Handle HTTP 429 with exponential backoff (skip next 3 cycles)

### Task 3: Register Flight Source
**Files:** `backend/src/config/sources.ts`, `backend/src/config/env.ts`, `backend/src/index.ts`, `.env`

1. Add flight source config to sources.ts
2. Add `OPENSKY_CLIENT_ID`, `OPENSKY_CLIENT_SECRET` to env.ts
3. Add placeholder values to `.env` and `.env.example`
4. Instantiate OpenSkyFetcher in index.ts and register with scheduler

### Task 4: FlightLayer Frontend Component
**Files:** `frontend/src/components/FlightLayer.tsx` (new)

Imperative BillboardCollection rendering:
1. Get viewer ref, create BillboardCollection on mount
2. Subscribe to `useLayerData('flights')`
3. On data update: clear all billboards, re-add from features
4. Each billboard: airplane SVG icon, rotated by `true_track` degrees, scaled by zoom level
5. Color by altitude band: ground=gray, low(<10k ft)=green, mid(10-30k)=blue, cruise(>30k)=white
6. Billboard `id` set to feature id for click handler
7. Return `null`

### Task 5: Aircraft SVG Icon
**Files:** `frontend/public/icons/aircraft.svg` (new)

Simple airplane silhouette SVG (top-down view):
- 32x32px, white fill, transparent background
- Points upward (north=0°), rotation applied by CesiumJS

### Task 6: Wire into App.tsx + ControlPanel
1. Add `<FlightLayer />` as child of `<Globe>` in App.tsx
2. Add flights toggle to ControlPanel (in "Live Data" group)
3. Add flight count to StatusBar

### Task 7: Docker Build + Test
1. Rebuild backend and nginx containers
2. Verify flights appear on globe (if credentials provided)
3. Verify fallback behavior without credentials (unauthenticated, slower updates)
4. Check WebSocket pushes flight data to connected clients
