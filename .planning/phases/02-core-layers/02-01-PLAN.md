# Plan 02-01: Globe Architecture Refactor + Layer Management UI

## Goal
Refactor Globe.tsx from monolithic Entity API to modular Primitive API architecture, wire up Zustand store + WebSocket hooks, and build a ControlPanel sidebar with layer toggles.

## Why First
All subsequent plans (flights, cameras, satellites) need the modular layer system to plug into. Without this refactor, adding 336K ALPR points via Entity API will crash the browser.

## Tasks

### Task 1: Refactor Globe.tsx to Modular Architecture
**Files:** `frontend/src/components/Globe.tsx`, `frontend/src/App.tsx`

Strip Globe.tsx down to ONLY the CesiumJS Viewer setup (scene config, lighting, camera, mouse handler). Remove all inline data fetching and Entity rendering. Globe.tsx becomes a shell that:
1. Creates the Resium `<Viewer>` with current settings (no timeline, no animation, OSM fallback, etc.)
2. Configures scene (lighting, fog, skyAtmosphere)
3. Sets default camera to US
4. Handles mouse move for coordinate display
5. Accepts `children` so layer components mount inside the Viewer

App.tsx becomes the orchestrator:
```tsx
<Globe>
  <EarthquakeLayer />
  <WeatherLayer />
  {/* Future: <FlightLayer />, <ALPRLayer />, etc. */}
</Globe>
<ControlPanel />
<StatusBar />
```

### Task 2: Refactor EarthquakeLayer.tsx to use PointPrimitiveCollection
**Files:** `frontend/src/components/EarthquakeLayer.tsx`

The existing EarthquakeLayer.tsx file already has the right structure but needs to be verified and wired up. It should:
1. Accept `viewer` prop (or use context/ref)
2. Use `useLayerData('earthquakes')` hook for data
3. Use `useLayerStore` to check visibility
4. Create `PointPrimitiveCollection` on mount, update points imperatively when data changes
5. Return `null` (no React DOM)
6. Color-code by magnitude using cesiumHelpers.ts functions

### Task 3: Refactor WeatherLayer.tsx to use GeoJsonDataSource
**Files:** `frontend/src/components/WeatherLayer.tsx`

Weather alerts are low-count polygons (~31), so GeoJsonDataSource is fine:
1. Accept `viewer` prop
2. Use `useLayerData('weather')` hook
3. Use `useLayerStore` for visibility
4. Load GeoJSON into a `GeoJsonDataSource` instance
5. Color polygons by severity
6. Return `null`

### Task 4: Wire up WebSocket + Zustand Store
**Files:** `frontend/src/hooks/useWebSocket.ts`, `frontend/src/hooks/useLayerData.ts`, `frontend/src/store/layerStore.ts`

The hooks and store already exist but aren't wired up. Connect them:
1. Verify `useWebSocket.ts` singleton pattern works with current Socket.IO setup
2. Verify `useLayerData.ts` fetches initial data via REST + subscribes to WebSocket updates
3. Expand `layerStore.ts` to include all Phase 2 layers (flights, alpr, speed_cameras, satellites) with `visible: false` initially
4. Add `lastUpdated` and `error` fields to layer state

### Task 5: Build ControlPanel Component
**Files:** `frontend/src/components/ControlPanel.tsx` (new)

Collapsible sidebar on the left with:
1. Dashboard title/logo at top
2. List of all layers with toggle switches
3. Each toggle shows: layer name, status icon (loading spinner / green dot / red X), entity count
4. Layers grouped by category: "Live Data" (earthquakes, weather, flights), "Surveillance" (ALPR, speed cameras), "Space" (satellites)
5. Styled dark theme to match the dashboard aesthetic
6. Mobile: collapses to hamburger menu below 768px
7. Uses `useLayerStore` for toggle state

### Task 6: Build StatusBar Component
**Files:** `frontend/src/components/StatusBar.tsx`

Replace the inline coordinate display with a proper status bar:
1. Bottom bar spanning full width
2. Shows: connection status (LIVE/OFFLINE), cursor coordinates, total entity counts per active layer, current UTC time
3. Uses `useLayerStore` for counts
4. Dark theme, monospace font
5. Replace current CoordinateDisplay.tsx usage

Rebuild Docker stack and verify:
