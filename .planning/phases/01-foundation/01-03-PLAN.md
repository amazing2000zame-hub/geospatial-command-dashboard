<!-- Backend WebSocket contract from Plan 01-02 -->
Socket.IO namespace: /layers
Events:
- Client emits 'subscribe' with layerId string -> server joins room `layer:{layerId}`
- Client emits 'unsubscribe' with layerId string -> server leaves room
- Server emits 'data' with { layerId: string, data: LayerFeatureCollection, timestamp: number }
- Server emits 'status' with { layerId: string, status: 'active'|'error', count: number, error?: string }

REST endpoints:
- GET /api/layers/earthquakes -> LayerFeatureCollection
- GET /api/layers/weather -> LayerFeatureCollection
- GET /api/health -> { status, timestamp, redis, uptime }

  <action>
    Build the frontend data pipeline and rendering layer.

    **1. frontend/src/hooks/useWebSocket.ts** - Socket.IO client from Pattern 8 in 01-RESEARCH.md:
    - Connect to '/layers' namespace with transports: ['websocket'], reconnection: true, reconnectionDelay: 1000, reconnectionAttempts: Infinity
    - Track connected state with useState
    - Expose subscribeLayer(layerId) and unsubscribeLayer(layerId) callbacks
    - Expose the socket ref for attaching data/status event listeners
    - Clean up on unmount: socket.disconnect()
    - Do NOT use localhost for connection URL -- in production, Socket.IO client auto-detects the origin (same as page URL through nginx proxy)

    **2. frontend/src/hooks/useLayerData.ts** - Combined REST + WebSocket data hook:
    - Takes layerId string parameter
    - On mount: fetch initial data from GET /api/layers/{layerId} (REST)
    - On mount: call subscribeLayer(layerId) to join Socket.IO room
    - Listen for 'data' events on socket: when layerId matches, update state with new FeatureCollection
    - Listen for 'status' events: update layer status in Zustand store
    - Return { data: LayerFeatureCollection | null, loading: boolean, error: string | null }
    - On unmount: unsubscribeLayer(layerId)
    - Handle both initial REST load failure (server not ready yet) and WebSocket disconnect gracefully

    **3. frontend/src/store/layerStore.ts** - Zustand store for layer UI state:
    - Create store with:
      - layers: Record<string, { visible: boolean, status: 'active'|'loading'|'error'|'disabled', count: number, error?: string }>
      - toggleLayer(layerId): flips visible boolean
      - setLayerStatus(layerId, status, count, error?): updates status info
    - Initialize with two layers: earthquakes (visible: true, status: 'loading') and weather (visible: true, status: 'loading')
    - Zustand 5 API: `create<State>()((set) => ({ ... }))`

    **4. frontend/src/components/EarthquakeLayer.tsx** - From Pattern 6 in 01-RESEARCH.md:
    - Props: { viewer: Cesium.Viewer | null, data: LayerFeatureCollection | null, visible: boolean, onSelectFeature: (feature) => void }
    - Use PointPrimitiveCollection (NOT Entity API -- Entity API leaks memory, see Pitfall 8)
    - Create collection once on mount: viewer.scene.primitives.add(new Cesium.PointPrimitiveCollection())
    - When data changes:
      - Call points.removeAll() (clears previous markers)
      - Loop through data.features where geometry.type === 'Point'
      - For each: points.add({ position: Cartesian3.fromDegrees(lon, lat), pixelSize: magnitudeToSize(mag), color: magnitudeToColor(mag), id: feature.properties.id })
      - Call viewer.scene.requestRender() (REQUIRED when requestRenderMode is true)
    - When visible changes: set points.show = visible
    - Handle click detection:
      - Use ScreenSpaceEventHandler on LEFT_CLICK
      - viewer.scene.pick(position) returns picked primitive
      - If picked primitive has an id matching an earthquake feature, call onSelectFeature with the full feature data
    - Store feature data in a Map<string, LayerFeature> ref for click lookup by id
    - Clean up on unmount: remove primitives collection, destroy event handler

    **5. frontend/src/components/WeatherLayer.tsx** - From Pattern 7 in 01-RESEARCH.md:
    - Props: { viewer: Cesium.Viewer | null, data: LayerFeatureCollection | null, visible: boolean, onSelectFeature: (feature) => void }
    - Use GeoJsonDataSource (OK for weather -- low count, infrequent updates)
    - CRITICAL: Filter data.features to only those with non-null geometry BEFORE loading (Pitfall 7 -- many NWS alerts lack polygons)
    - Create GeoJsonDataSource('weather-alerts'), call ds.load(filteredGeoJSON) with:
      - stroke: Cesium.Color.WHITE, strokeWidth: 2
      - fill: Cesium.Color.YELLOW.withAlpha(0.3)
      - clampToGround: true
    - After load: iterate entities, apply severity-based colors using severityToColor():
      - Read entity.properties.severity.getValue() for each entity
      - Set entity.polygon.material = new ColorMaterialProperty(severityToColor(severity))
    - Add dataSource to viewer.dataSources
    - When data changes: remove old dataSource (with destroy=true), create and add new one
    - Use suspendEvents()/resumeEvents() around bulk entity updates if modifying multiple entities
    - When visible changes: set dataSource.show = visible
    - Click handling: CesiumJS infoBox handles Entity clicks automatically for GeoJsonDataSource entities.
      Set entity.description to HTML string with alert details (event, severity, headline, areaDesc, instruction).
      Also call onSelectFeature for the custom popup.
    - Call viewer.scene.requestRender() after loading
    - Clean up on unmount: viewer.dataSources.remove(ds, true)

    **6. frontend/src/components/StatusBar.tsx** - Connection and count display:
    - Position absolute bottom-right overlay
    - Show: WebSocket connection status (green dot = connected, red = disconnected)
    - Show: "Earthquakes: {N}" count from Zustand store
    - Show: "Weather Alerts: {N}" count from Zustand store
    - Style: dark semi-transparent background, white text, monospace font, small font size
    - Update reactively from Zustand store

    **7. frontend/src/components/LayerToggle.tsx** - Simple layer toggle controls:
    - Position absolute top-left overlay
    - For each layer in Zustand store: render a toggle button/checkbox
    - Show layer name, status indicator (green/yellow/red dot), and count
    - On toggle: call store.toggleLayer(layerId)
    - Style: dark semi-transparent panel, compact

    **8. frontend/src/components/DetailPopup.tsx** - Click-to-detail popup:
    - Position absolute, appears near click location or in a fixed panel
    - Receives selected feature (LayerFeature | null) as prop
    - If earthquake (layer === 'earthquakes'):
      - Show: Magnitude (large, colored), location (place), depth in km, time (formatted), USGS link
    - If weather (layer === 'weather'):
      - Show: Event type (headline), severity (colored badge), affected area, description, instruction, effective/expires times
    - Close button to dismiss
    - Style: dark panel with white text, max-width 350px

    **9. frontend/src/App.tsx** - Wire everything together:
    - Import Globe, EarthquakeLayer, WeatherLayer, StatusBar, LayerToggle, DetailPopup
    - Import useWebSocket, useLayerData, useLayerStore
    - Initialize WebSocket connection
    - Call useLayerData('earthquakes') and useLayerData('weather') to get live data
    - Track selectedFeature state for DetailPopup
    - Get viewer ref from Globe (Globe should expose viewer via callback prop or ref)
    - Read layer visibility from Zustand store
    - Render:
      ```
      <Globe onViewerReady={setViewer}>
        {/* Layers rendered as siblings, not children of Viewer -- they use imperative API */}
      </Globe>
      <EarthquakeLayer viewer={viewer} data={quakeData} visible={layers.earthquakes.visible} onSelectFeature={setSelected} />
      <WeatherLayer viewer={viewer} data={weatherData} visible={layers.weather.visible} onSelectFeature={setSelected} />
      <StatusBar />
      <LayerToggle />
      <DetailPopup feature={selectedFeature} onClose={() => setSelected(null)} />
      ```
    - Note: EarthquakeLayer and WeatherLayer render null (no DOM output) -- they operate imperatively on the CesiumJS viewer. They can be rendered anywhere in the React tree as long as they receive the viewer prop.
