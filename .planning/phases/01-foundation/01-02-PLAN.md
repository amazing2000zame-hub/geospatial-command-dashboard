<objective>
Build the complete Fastify 5 backend with Socket.IO real-time updates, Redis caching, and two proof-of-concept data fetchers (USGS earthquakes, NWS weather alerts).

Purpose: This backend serves as the data pipeline for all current and future data layers. The BaseFetcher pattern, caching service, and WebSocket broadcast system established here will be reused for every additional data source in Phases 2-4. Getting this right in Phase 1 means adding a new data source in future phases is just writing one fetcher file.

Output: A fully functional backend that polls USGS and NWS APIs on schedule, normalizes responses to unified GeoJSON, caches in Redis with TTL, broadcasts updates via Socket.IO rooms, and serves REST endpoints for initial data load.

    Implement the data fetcher pattern and two proof-of-concept fetchers.

    **1. backend/src/fetchers/BaseFetcher.ts** - Abstract class from Pattern 5 in 01-RESEARCH.md:
    - Abstract class BaseFetcher with:
      - Abstract readonly sourceId: string
      - Abstract readonly displayName: string
      - Abstract readonly defaultInterval: string (cron expression)
      - Abstract readonly cacheTTL: number (seconds)
      - Constructor takes (cache: CacheService, io: Namespace)
      - Abstract fetchRaw(): Promise<unknown>
      - Abstract normalize(raw: unknown): LayerFeatureCollection
      - execute() method that:
        1. Checks if cache is still fresh (cache.get returns non-null) -- if fresh, skip fetch and return early
        2. Calls fetchRaw() to get raw API response
        3. Calls normalize(raw) to convert to unified GeoJSON
        4. Calls cache.set(sourceId, normalized, cacheTTL)
        5. Emits to Socket.IO room: io.to(`layer:${sourceId}`).emit('data', { layerId, data, timestamp })
        6. Emits status update to ALL clients: io.emit('status', { layerId, status: 'active', count })
        7. Catches errors: logs, calls cache.setError(), emits error status, does NOT rethrow (other fetchers must continue)

    **2. backend/src/fetchers/usgs.ts** - USGS earthquake fetcher (EQKE-01):
    - Extends BaseFetcher
    - sourceId = 'earthquakes'
    - displayName = 'USGS Earthquakes'
    - defaultInterval = '*/60 * * * * *' (every 60 seconds)
    - cacheTTL = 55 (slightly less than interval to ensure fresh data on next poll)
    - fetchRaw(): fetch() the USGS URL (from env or default constant)
      - URL: https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_day.geojson
      - No auth headers needed
      - Parse as JSON, return the raw GeoJSON FeatureCollection
    - normalize(raw): Convert USGS GeoJSON to unified LayerFeatureCollection:
      - Map each feature:
        - id: feature.id (USGS provides stable IDs like "ci40000001")
        - layer: 'earthquakes'
        - label: `M${feature.properties.mag}` (e.g., "M4.2")
        - timestamp: Math.floor(feature.properties.time / 1000) (USGS uses ms, normalize to seconds)
        - category: feature.properties.type || 'earthquake'
        - severity: clamp(feature.properties.mag / 10, 0, 1) (normalize 0-10 Richter to 0-1)
        - Preserve original properties: mag, place, url, title, depth (from geometry.coordinates[2])
      - Keep geometry as-is (already GeoJSON Point with [lon, lat, depth])
      - Set metadata: source 'usgs', fetchedAt Date.now(), count, nextUpdate

    **3. backend/src/fetchers/nws.ts** - NWS weather alerts fetcher (WTHR-01):
    - Extends BaseFetcher
    - sourceId = 'weather'
    - displayName = 'NWS Weather Alerts'
    - defaultInterval = '*/120 * * * * *' (every 2 minutes)
    - cacheTTL = 110
    - fetchRaw(): fetch() the NWS URL with REQUIRED headers:
      - 'User-Agent': env.NWS_USER_AGENT (e.g., 'GeospatialDashboard/1.0 (homelab@localhost)')
      - 'Accept': 'application/geo+json'
      - URL: https://api.weather.gov/alerts/active
      - Without User-Agent header, NWS returns 403 (see Pitfall 6 in research)
    - normalize(raw): Convert NWS GeoJSON to unified LayerFeatureCollection:
      - CRITICAL: Filter out features where geometry is null (see Pitfall 7 -- many NWS alerts are county-based without polygon geometry, typically 30-50% of alerts)
      - Map each feature with non-null geometry:
        - id: feature.properties.id (NWS provides IDs like "NWS-IDP-PROD-12345")
        - layer: 'weather'
        - label: feature.properties.event (e.g., "Tornado Warning")
        - timestamp: Math.floor(new Date(feature.properties.sent).getTime() / 1000)
        - category: feature.properties.event
        - severity: map string to 0-1 (extreme=1.0, severe=0.75, moderate=0.5, minor=0.25, unknown=0.1)
        - Preserve: areaDesc, headline, description, instruction, effective, expires, urgency, certainty
      - Keep geometry as-is (Polygon or MultiPolygon)

    **4. backend/src/services/scheduler.ts** - Cron scheduler with staggered startup:
    - Export function startScheduler(fetchers: BaseFetcher[], sourceConfigs: SourceConfig[])
    - For each enabled source config:
      - Find the matching fetcher by sourceId
      - Register a node-cron job with the source's interval expression
      - On trigger: call fetcher.execute()
    - Staggered initial fetch (Pitfall 5: thundering herd):
      - On startup, do NOT run all fetchers at t=0
      - Instead, stagger initial fetches with random jitter: for each fetcher, setTimeout with random delay 0-5 seconds
      - This prevents all 17 APIs (in future phases) from being hit simultaneously on container restart
    - Return an array of cron jobs for graceful shutdown
    - Log each scheduled job: "[scheduler] Registered {displayName} every {interval}"

    **5. backend/src/index.ts** - Wire everything together:
    - Import loadEnv, createServer, startScheduler, sourceConfigs, USGSFetcher, NWSFetcher
    - Load config
    - Create server (gets back fastify, io, cache, layerNs)
    - Instantiate fetchers: new USGSFetcher(cache, layerNs), new NWSFetcher(cache, layerNs)
    - Start scheduler with fetchers and configs
    - Handle SIGINT/SIGTERM: stop cron jobs, await cache.disconnect(), await fastify.close()
    - Log: "[startup] Geospatial Dashboard backend running on port {port}"
